# 移动端Page文章渲染实现
`@src\views\Article\components\MarkdownViewer.vue` 你参考这个文件，如果我要实现一个专门针对移动端的md渲染器，
于`@src\views\Article\components\MobileMarkdownViewer.vue`中，
移动端的文章都是pulic，直接和前者走一样的public接口，但是路由到/markdown/mobile/{id}中(注册对应的组件)，你可以实现吗?

**重要**：
1.兼容大部分的移动端设备，特别是**手机**
2.不要编辑`MobileMarkdownViewer.vue`外的其它文件(可以看，需要编辑就反问我)

## 样式
学习MarkdownViewer的(浅蓝白色调，但是稍暗一点)

顶部topbar：
1.左侧栏和右侧栏都不需要顶部导航左侧还是渲染`老鱼人文档中心`，右侧则渲染item(首页(目前只渲染这个))
2.topbar中的字体都渲染成蓝色，右边item带适当厚度的下划线，topbar粘贴在顶部，做一点透明(如:`MarkdownViewer.vue`)
点击首页：触发路由返回首页`/mobile-home`

中部核心：
1.核心markdown渲染部分，学习并改良`MarkdownViewer.vue`中的样式，如代码块渲染，标题渲染等等，添加合适的间距，字体不要太小方便阅读
2.要保留作者名称，字数，发布时间这些元素(和MarkdownViewer渲染的一样，图标也用那几个，调整下大小就行了)
3.宽度为视口宽度，高度不限(取决于文章长短)，用户下滑继续阅览

一个小工具(网上的文章渲染都很常见)：
右下角一个粘贴住的圈圈，圈圈内有一个向上的蓝色箭头，圈圈外边框为蓝色(和文章阅读进度(或者说长度)刚好对应)
点击箭头就回到顶部，然后到最底部时变得比较透明但还是有区分度(让用户可以看到底部内容且能点击箭头组件回到顶部)
注意因为是移动端，这个工具不要做的太大，遮挡太多内容影响观感。
